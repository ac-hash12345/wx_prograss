 
## 一、课程设计题目
基于微信小程序的网上商城系统（含PHP+MySQL后台管理）

## 二、设计目标
1. 掌握微信小程序核心开发技术（页面、组件、API、路由、数据绑定、网络请求等）。
2. 掌握PHP后端接口开发、MySQL数据库设计、前后端数据交互。
3. 实现商城核心业务功能，后台完成完整数据增删改查（CRUD）。
4. 覆盖小程序主流知识点，符合课程设计实战要求。

## 三、功能模块划分
### （一）微信小程序端（前端）
1. **用户模块**：用户授权登录、个人中心、用户信息展示
2. **商品模块**：商品列表、商品详情、商品分类、搜索商品
3. **购物车模块**：加入购物车、购物车列表、修改数量、删除商品
4. **订单模块**：创建订单、订单列表、订单详情
5. **基础功能**：下拉刷新、上拉加载、图片预览、消息提示

### （二）PHP后台管理端（Web）
1. **管理员登录**
2. **商品管理**：商品增删改查、分类管理
3. **订单管理**：订单查询、修改订单状态
4. **用户管理**：查看小程序用户信息

## 四、微信小程序端设计要求（覆盖核心知识点）
### 1. 页面结构要求
- 至少包含**6个主页面**：首页、商品列表、商品详情、购物车、个人中心、订单列表
- 合理使用**app.json全局配置**：导航栏标题、颜色、底部tabBar

### 2. 核心组件使用
必须使用小程序官方基础组件：
- `<view>`、`<text>`、`<image>` 布局渲染
- `<swiper>` 轮播图（首页商品轮播）
- `<scroll-view>` 滚动区域
- `<button>` 按钮（登录、加购、下单）
- `<input>` 搜索框
- `<checkbox>` 购物车选择
- `<navigator>` 页面跳转

### 3. 核心API与逻辑
- **wx.login** / **wx.getUserProfile**：微信授权登录
- **wx.request**：发起网络请求，与PHP后端接口交互
- **wx.showToast / wx.showModal**：消息提示、确认弹窗
- **wx.setStorage / wx.getStorage**：本地缓存（存储用户信息、购物车）
- 下拉刷新 `onPullDownRefresh`、上拉加载 `onReachBottom`
- 页面生命周期：`onLoad`、`onShow`、`onReady`

### 4. 数据绑定与事件
- 双向数据绑定 `{{}}`
- 点击事件 `bindtap`、表单绑定 `bindinput`
- 列表渲染 `wx:for`
- 条件渲染 `wx:if`

## 五、PHP+MySQL后台设计要求
### 1. 数据库设计（MySQL）
必须创建4张核心表：
1. `admin` 管理员表（id、账号、密码）
2. `user` 小程序用户表（id、微信昵称、头像、openid、创建时间）
3. `goods` 商品表（id、名称、价格、封面图、详情、库存、分类id、创建时间）
4. `orders` 订单表（id、订单号、用户id、商品信息、总价、订单状态、创建时间）

### 2. 后台接口要求（PHP）
所有接口返回**JSON格式**，小程序通过HTTP调用：
1. 用户登录接口（接收openid，返回用户信息）
2. 商品列表/详情接口
3. 购物车、创建订单、订单列表接口

### 3. 后台管理功能（Web增删改查）
#### ① 商品管理
- **增**：添加商品（名称、价格、图片、详情）
- **删**：删除商品
- **改**：修改商品信息、价格、库存
- **查**：分页查询所有商品、搜索商品

#### ② 订单管理
- **查**：查看所有订单、按状态筛选
- **改**：修改订单状态（待付款→待发货→已完成）

#### ③ 用户管理
- **查**：查看小程序注册用户列表
- **删**：删除违规用户

#### ④ 管理员管理
- 登录、退出、修改密码

## 六、开发技术要求
1. **前端**：微信小程序原生开发（WXML + WXSS + JS + JSON）
2. **后端**：PHP（原生或简单框架）+ MySQL
3. **服务器**：phpStudy / XAMPP 本地环境
4. **数据交互**：POST/GET请求，JSON格式数据
